1. 인터페이스
  - 특징
    인터페이스란 클래스가 구현해야 하는 멤버들을 정의
    인터페이스는 클래스의 일종이 아니며, 클래스에 대한 제약 조건을 명시    
    클래스가 인터페이스를 구현할 경우, 모든 인터페이스 멤버를 구현 
    인터페이스는 다중 상속을 지원

  - 장점
    코드의 재사용성
    다중 상속 제공
    유연한 설계

2. 열거형
 - 장점
    가독성
    자기 문서화
    스위치 문과의 호환

3. 값형과 참조형
 - 값형 : 값을 직접 저장
 - 참조형 : 메모리 주소 저장

4. 델리게이트
  -특징
    델리게이트(delegate)는 메서드를 참조하는 타입
    다른 프로그래밍 언어에서는 함수 포인터라는 용어
    델리게이트를 이용하면 메서드를 매개변수로 전달하거나 변수에 할당 가능
  -Func 과 Action
    return 값이 있는 func vs return 값이 없는 Action

5. 람다
  -특징
    익명 메서드를 만드는 방법
    메서드의 이름 없이 메서드를 만들 수 있습니다.
    람다는 델리게이트를 사용하여 변수에 할당하거나, 메서드의 매개변수로 전달
  -예시
    (parameter_list) => expression
